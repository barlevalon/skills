# Workflow map

Use this when you are not sure which skill to choose.

## Starting a feature

- Fuzzy idea ā†’ `write-a-prd`
- PRD needs an implementation plan ā†’ `prd-to-plan`
- Plan needs pressure-testing ā†’ `grill-with-docs`
- Unsure about UX/state/design ā†’ `prototype`

## Fixing or changing code

- Failure is unclear ā†’ `diagnose`
- Behavior change is clear ā†’ `tdd`
- Need broader context first ā†’ `zoom-out`
- Need branch/worktree help ā†’ `worktrunk`

## Reviewing work

- Strict maintainability review ā†’ `thermo-nuclear-code-quality-review`
- Architecture feels tangled ā†’ `improve-codebase-architecture`
- Need a visual explanation ā†’ `plannotator-visual-explainer`
- Want to learn from rejected plans ā†’ `plannotator-compound`
- Turning a goal into an execution package ā†’ `plannotator-setup-goal`

## Writing and communicating

- Docs work ā†’ `documentation-system`
- Commit message ā†’ `caveman-commit`
- Very concise agent responses ā†’ `caveman`
- Continue in another session ā†’ `handoff`

## Releasing

- Changelog, version choice, release validation ā†’ `release-prep`

## Finding more

- Need a skill this bundle does not have ā†’ `find-skills`
